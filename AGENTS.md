## 项目定位

这是一个基于 Vue 3、Vite、TypeScript 和高德地图 Web JS API 的私有租房地图工具。当前主应用集中在 `src/App.vue`，样式集中在 `src/styles/rental-map.scss`，没有拆分成多页面或多模块架构。

协作时必须以“当前生效代码”作为事实来源，不要把旧样式遗留类名、历史实现思路或未上线想法写成既有能力。

## 当前真实实现

### 地图与初始化

- 使用 `@amap/amap-jsapi-loader` 加载高德 JS SDK 2.0。
- 从 `VITE_AMAP_KEY` 读取 Key；若存在 `VITE_AMAP_SECURITY_JS_CODE`，会写入 `window._AMapSecurityConfig.securityJsCode`。
- 地图初始化参数固定为：
  - `viewMode: '2D'`
  - `zoom: 13`
  - `resizeEnable: true`
  - `animateEnable: false`
- 默认添加 `AMap.Scale` 和右下角 `AMap.ToolBar({ position: 'RB' })`。
- 初始化失败、缺少 Key、插件加载失败都会更新 `mapStatus` / `hasMapError` 并输出统一日志。

### 默认范围与视觉基线

- 默认中心点为 `DEFAULT_CENTER`，默认找房范围来自 `DEFAULT_AREA` 多边形。
- 地图底图必须保持高德默认样式，不要新增 `mapStyle` 或全局滤镜。
- 私有范围通过描边 + 范围外半透明遮罩表现；范围内不加填充色。
- 地铁重点展示层固定为上海 11 号线嘉定北到南翔一段的站点标签，不额外绘制线路覆盖线。

### 页面布局

- 当前生效布局是“全屏地图 + 悬浮面板”，根容器是 `.map-workspace`，地图容器是 `.map-container`。
- 搜索面板、工具面板、状态面板、确认面板、标记面板和标记弹窗都以绝对定位覆盖在地图上。
- 移动端下工具按钮固定到底部区域，标记面板和状态面板会重新排布，但页面仍然不能出现额外滚动。
- `rental-map.scss` 中仍保留了 `.rental-map`、`.rental-map__sidebar`、`.map-toolbar` 等旧布局类名；这些目前不是主模板的事实来源，后续改版不要按这些遗留类继续扩展。

### 搜索

- 顶部搜索框支持地点搜索、历史关键词展示、历史关键词点击重搜、搜索结果清除。
- 搜索优先走高德 REST `place/around` 分页检索：
  - 以当前范围几何中心为圆心。
  - 半径由当前多边形顶点到中心的最大距离推导，并限制在 `3000m ~ 50000m`。
  - 结果会再经过当前多边形过滤。
- 若范围周边搜索无结果或失败，会回退到：
  - 高德 REST `place/text`
  - 再回退到 `AMap.PlaceSearch`
- 搜索结果会：
  - 计算最近 11 号线站点与距离。
  - 按最近地铁距离排序。
  - 渲染编号 Marker。
  - 支持列表选中后只居中地图，不修改当前缩放级别。
- 可以基于搜索结果直接创建房源 Marker，并预填优点/缺点文案。

### Marker 管理

- 点击地图合法位置可创建临时 Marker，并打开弹窗编辑：
  - `pros`
  - `cons`
  - `isRejected`
- 点击已有 Marker 会打开编辑弹窗。
- Marker 删除支持：
  - 左上角列表删除
  - 地图自定义 Marker 右上角删除按钮
- Marker 使用自定义 HTML 内容渲染：
  - 优点和缺点都完整展示，不做省略号截断。
  - `cons` 使用红色强调。
  - `isRejected` 时卡片背景和边框切换为浅红色方案。
- 保存前会校验位置存在且坐标仍在当前范围内。

### 范围选择

- 快捷键 `R` 或工具按钮可进入框选模式。
- 使用 `AMap.MouseTool.rectangle` 完成矩形框选。
- 框选后只生成 `pendingAreaPath`，需要显式确认才会真正替换当前范围。
- 确认新范围时会：
  - 更新 `areaPath`
  - 移除范围外 Marker
  - 清理待确认框选、路线和 POI
  - 重绘遮罩、边框、地铁标签和房源 Marker
  - 重新聚焦到新范围

### POI 与路线

- 对当前选中 Marker 支持搜索附近：
  - `地铁站`
  - `公交站`
- 附近搜索通过 `AMap.PlaceSearch.searchNearBy` 完成，参数固定为：
  - 半径 `1600`
  - `pageSize: 8`
- 支持两种路线规划入口：
  - 点击附近 POI
  - 进入“选点路线”模式后直接点地图目标点
- 步行路线通过 `AMap.Walking` 完成，结果写入 `#route-panel`。
- 清除路线时会同时清空地图路线覆盖物和 `#route-panel` HTML。

### 导入导出与本地存储

- 导出文件名格式为 `rental-map-YYYY-MM-DD.json`。
- 当前导出结构固定为 `version: 1`，包含：
  - `name`
  - `exportedAt`
  - `area.bbox`
  - `area.path`
  - `markers`
- 导入时会做严格结构校验，并兼容旧字段：
  - `title -> pros`
  - `note -> cons`
- 若 `localStorage` 不可用，页面会明确提示用户改用“保存 JSON / 导入 JSON”。
- 本地存储当前仅保存：
  - Marker 数据
  - 搜索历史
- 当前代码不会把范围配置自动持久化到 `localStorage`；范围持久化依赖 JSON 导出导入。

### 本地离线构建

- `pnpm build:local` 会在 `dist-local/` 生成内联 CSS/JS 的单文件版本 `free-map.html`。
- `pnpm build:mac` 会进一步打包为 `free-map-mac.zip`。
- 若修改了构建产物组织方式、静态资源引用方式或下载说明，需要同步检查 `scripts/inline-local-build.mjs`。

## 关键文件

- `src/App.vue`：核心逻辑与模板，包含地图初始化、搜索、Marker、导入导出、路线和日志。
- `src/styles/rental-map.scss`：当前页面的主要视觉和响应式布局。
- `src/styles/main.scss`：全局入口与基础排版。
- `src/env.d.ts`：环境变量与 `window._AMapSecurityConfig` 类型声明。
- `scripts/inline-local-build.mjs`：本地单文件构建脚本。
- `package.json`：开发、构建、本地打包脚本定义。

## 界面样式硬约束

### 最高优先级

1. `ibot2-frontend` 的所有布局和样式都必须从 `ibn-knowledge-factory-fe` 复刻，不要自己添加局部样式或者 `ibn-knowledge-factory-fe` 没有的样式，除 Vue 2 到 Vue 3 的兼容代码外。

### 当前项目约束

1. 当前项目核心交互模式必须保持为“地图全屏 + 悬浮按钮 + 按需悬浮面板”。
2. 不要新增侧边栏、普通顶部导航、说明页、营销页、卡片式整页布局，不能挤占地图视口。
3. 样式修改优先复用 `rental-map.scss` 现有颜色、边框、阴影、圆角、字号和间距体系。
4. 不要因为局部需求引入新的孤立视觉风格。
5. 不要把遗留未使用类名重新当作新的布局入口，除非先确认模板已恢复引用。
6. 移动端调整不能通过撑高页面来换布局，地图区域必须始终占满视口。

## Debugging 与改动原则

你不仅是代码生成器，更是资深 Debugging 专家。处理问题时必须遵循下面流程。

### 第一步：根因追溯

- 禁止看到现象就直接改代码。
- 先根据报错信息、日志、调用路径和现有实现，至少列出 2 个可能触发场景。
- 判断是否需要联网检索外部资料：
  - 高德 SDK / REST API 行为
  - 浏览器兼容性
  - Vite / Vue / TypeScript 新版本行为
- 如果需要检索，先整理资料，再更新根因判断。
- 必须解释“为什么当前代码逻辑会导致这个问题”，不能只描述表象。

### 第二步：增强可观测性

- 先判断现有日志是否足以定位问题。
- 如果不足，必须先补充日志，再做功能修复。
- 日志至少覆盖：
  - 关键入参和状态
  - 函数进入/退出
  - 插件加载结果
  - 异步回调结果
  - 失败分支命中原因
- 不要移除已有日志，除非有等价或更强的可观测性替代。

### 第三步：结构化修复

- 修复代码必须带完整防御性编程：
  - 空值检查
  - 类型与边界校验
  - 失败后的状态回收
  - UI 状态一致性维护
- 解释修复逻辑如何消除第一步中列出的隐患，而不是只让个别样例通过。
- 不要用启发式规则硬补特殊分支；如果无法做成完整逻辑覆盖，必须明确给出方案选项。
