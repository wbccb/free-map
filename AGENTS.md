## 项目当前已实现功能

这是一个基于 Vue 3、Vite、TypeScript 和高德地图 Web JS API 的租房找房私有地图工具。

### 地图基础能力

- 使用 `@amap/amap-jsapi-loader` 加载高德地图 JS SDK 2.0。
- 从 `VITE_AMAP_KEY` 读取高德 Web JS API Key。
- 可选读取 `VITE_AMAP_SECURITY_JS_CODE` 并写入 `window._AMapSecurityConfig.securityJsCode`。
- 地图使用 2D 模式、高德默认底图样式、比例尺控件和右下角工具条控件。
- 默认中心点锁定到上海 11 号线嘉定北站到南翔站之间，默认范围是这段线路相关站点周边约 3km 的多边形。
- 叠加 11 号线站点重点展示层：嘉定北、嘉定西、白银路、嘉定新城、马陆、陈翔公路、南翔站点使用放大标签展示；线路本身使用高德默认底图显示，不额外绘制覆盖线。
- 顶部提供地点搜索栏，优先使用高德 REST Place Around API 以当前私有范围中心和范围半径做周边检索，再用当前范围多边形过滤结果；周边检索无可用结果时回退到 REST Place Text API 和 `AMap.PlaceSearch`。搜索成功后把关键词存入 `localStorage`，输入框聚焦时展示历史关键词，点击历史关键词会直接重新搜索。搜索结果列表展示与地图 Marker 对应的编号、地点名称、最近 11 号线站点和距离，不展示地址行，并按距离最近地铁站从近到远排序；搜索结果会在地图上渲染明显编号 Marker。点击列表项或结果 Marker 后，地图只居中到该地点，不改变当前缩放级别，当前聚焦结果的列表项高亮，地图编号 Marker 改色并放大。搜索栏提供“清除”按钮，只清除搜索结果列表、搜索结果 Marker、搜索关键词和搜索选中状态，不删除房源 Marker。
- 地图初始化失败、缺少 Key、SDK 加载失败时会更新页面状态并输出日志。

### 全屏地图 + 悬浮按钮模式

- 主界面是全屏地图：`.map-workspace` 固定铺满视口，`.map-container` 高度为 `100vh`。
- 所有操作入口都是覆盖在地图上的悬浮按钮或悬浮面板，不允许再引入占据页面布局的侧边栏、顶部栏或普通文档流面板。
- 默认可见的是右上角工具按钮组；状态面板只在缺少 Key、发生错误、处于交互模式或存在待确认范围时显示。
- 标记列表默认常驻左上角悬浮显示，展示全部 Marker 集合。
- 移动端工具按钮固定在底部区域，地图仍然保持全屏，不产生额外页面滚动。

### 私有地图范围

- 支持“框选范围 R”进入矩形框选模式。
- 使用 `AMap.MouseTool.rectangle` 绘制范围。
- 框选完成后生成 `pendingAreaPath`，需要点击“确定”才会锁定范围。
- 确认范围时：
  - 更新当前私有地图范围。
  - 移除范围外的已有标记。
  - 重绘范围外半透明遮罩和范围边框；范围内保持高德默认底图颜色。
  - 聚焦到新范围。
- 支持取消当前框选范围。
- 支持“回到范围”重新聚焦当前私有地图范围。

### 标记管理

- 支持直接点击地图有效经纬度点位创建临时 Marker，并打开标记弹窗填写优点、缺点和“垃圾小区，不能租用”勾选项；优点和缺点都不是必填项，勾选项默认不勾选。新增、编辑、删除 Marker 后自动保存到 `localStorage`，下次加载优先恢复本地 Marker 数据。
- 保存标记前会校验：
  - 点击位置存在。
  - 坐标位于当前私有地图范围内。
- 支持在标记列表中选择标记，地图只会居中到该标记，不改变当前缩放级别。
- 支持从标记列表删除标记，也支持点击地图 Marker 右上角删除按钮删除标记；点击地图 Marker 本体会打开编辑弹框重新编辑优点、缺点和是否不能租用；删除当前选中标记时会自动切换到剩余第一个标记或清空选中状态。
- 标记使用自定义 HTML marker 渲染，优点和缺点通过 `textContent` 写入并完整换行显示，不使用省略号截断；缺点在地图 Marker 和左上角列表中使用红色文字显示；勾选“垃圾小区，不能租用”后，地图 Marker 背景使用浅红色。

### 附近搜索和路线规划

- 标记列表面板中支持对当前选中标记搜索附近“地铁站”和“公交站”。
- 使用 `AMap.PlaceSearch.searchNearBy`，搜索半径为 1600 米，单次最多 8 条。
- 搜索结果会渲染为地图 POI marker 和面板内 POI 列表。
- 点击 POI 可使用 `AMap.Walking` 规划从当前选中标记到 POI 的步行路线。
- 支持“选点路线”：进入模式后在地图上点击目的地，规划从当前选中标记到该点的步行路线。
- 支持清除路线，并清空 `#route-panel` 内容。

### JSON 导入导出

- 支持导出当前地图配置为 `rental-map-YYYY-MM-DD.json`。
- 导出结构版本为 `version: 1`，包含：
  - `name`
  - `exportedAt`
  - `area.bbox`
  - `area.path`
  - `markers`
- 支持导入 JSON 配置。
- 导入时会做结构化校验：
  - 顶层必须是对象。
  - `version` 必须为 `1`。
  - `area.path` 至少 3 个合法经纬度点，或提供合法 `area.bbox`。
  - `markers` 必须是数组。
  - 每个 marker 必须包含非空 `id` 和合法坐标。
  - 每个 marker 支持字符串 `pros`、字符串 `cons` 和布尔值 `isRejected`；`pros` 和 `cons` 可以为空，`isRejected` 默认 `false`。
  - 为兼容旧数据，导入或读取本地 Marker 时会把旧字段 `title` 迁移为 `pros`，把旧字段 `note` 迁移为 `cons`。
  - marker 坐标必须位于导入范围内。
  - marker id 不能重复。
- 导入成功后会清除路线、POI、待确认范围和标记弹窗，替换当前范围和标记，并聚焦到导入范围。

### 快捷键和交互模式

- `R`：进入框选范围模式。
- `Esc`：取消当前交互。
- 输入框或文本域聚焦时不会响应快捷键。
- 当前交互模式包括：`idle`、`draw-area`、`pick-route`。

### 日志和可观测性

- 使用 `createLogger('RentalMap')` 统一输出日志。
- 已覆盖的关键日志包括：
  - SDK 加载开始、成功和失败。
  - 地图创建完成。
  - 框选开始、完成、确认和取消。
  - 地图移动开始和结束耗时。
  - JSON 导入、导出、导入失败。
  - 标记弹窗打开、标记保存失败、标记新增、删除和选择。
  - 地点搜索开始、完成和结果选择。
  - 附近搜索开始、完成和失败。
  - 路线规划开始、完成和失败。

## 界面样式原则

1. 当前项目的核心界面模式是“地图全屏 + 悬浮按钮 + 按需悬浮面板”。
2. 不要新增侧边栏、普通顶部导航、页面卡片布局、营销页、说明页或任何会挤占地图空间的布局。
3. 不要设置 `mapStyle` 改浅或重绘高德底图颜色；底图必须保持高德默认地图颜色。
4. 私有范围内不要增加填充色或地图整体滤镜；范围外允许使用半透明遮罩强化边界识别。
5. 样式修改优先复用当前 `rental-map.scss` 已有颜色、边框、圆角、阴影、字号和间距。
6. 不要为了局部需求添加孤立的新视觉风格。
7. 如果未来接入 `ibn-knowledge-factory-fe` 作为样式复刻来源，`ibot2-frontend` 的布局和样式必须从 `ibn-knowledge-factory-fe` 复刻，不要自行添加 `ibn-knowledge-factory-fe` 没有的局部样式，除 Vue 2 到 Vue 3 的兼容代码外。
